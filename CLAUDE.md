# 작업 가이드 (Antigravity 에이전트 필독)

이 문서는 "클럽 비용 정산기" 웹앱 작업 시 **반드시 지켜야 할 규칙**을 정리한 것입니다.
새 작업을 시작하기 전에 이 문서를 먼저 읽으세요.

---

## 1. 절대 규칙 (Hard Constraints)

0. **모든 계산/숫자 표시 관련 버그는 무관용(zero-tolerance) 원칙**
   - 정산 금액은 실제 회사 비용 처리와 직결됨 — "결과값은 맞는데 안내 문구/팝업/표시만 틀린" 경우도
     동일한 심각도로 취급할 것 (예: v1.6.76 자부담 차액 팝업 버그).
   - 계산 관련 코드를 수정하거나 새로 만들 때는 **CALCULATION_SPEC.md / 계산식_명세서.md의 예시값으로
     반드시 직접 대조 검증**.
   - 이벤트 중복 호출, 상태값(`AppState.xxx`) 갱신 순서 등 **부수효과가 있는 로직은 특히 꼼꼼히 점검**
     — "수정 전 값" 같은 기준점은 매번 변하는 상태값이 아니라 고정된 자동계산값을 사용해야 함.
   - 화면에 보이는 모든 금액/비율/카운트(정산 결과, 엑셀, 차트, 명부 카운트, 잔여 예산 등)는
     "어떤 변수에서 왔는지"를 코드로 직접 확인하기 전까지 맞다고 가정하지 말 것.

1. **모든 답변/커밋 메시지 설명은 한국어로 작성** (코드 식별자/주석은 기존 스타일 유지).
2. **계산 로직 & 엑셀 매핑은 절대 임의 변경 금지**
   - 기준 문서: [`CALCULATION_SPEC.md`](CALCULATION_SPEC.md), [`계산식_명세서.md`](계산식_명세서.md)
   - `SettlementCalculator.calculate`, `calculateSelfPayPerPerson`, `generateExcelFile`의 셀 매핑(K4~K30, L12~L25 등)은
     **사용자가 명시적으로 요청하지 않는 한 절대 수정하지 않음**.
   - 템플릿 셀 좌표가 바뀌는 경우에도 계산식 자체는 그대로 두고 `setCellValue` 의 ref만 수정.
3. **웹 전용 배포 — APK 자산 동기화 금지**
   - `app/src/main/assets/` 디렉터리에 변경 파일을 복사/동기화하지 않음 (과거에는 했었지만 현재는 GitHub Pages 웹앱만 운영).
4. **권한 모델 변경 시 [`PERMISSIONS.md`](PERMISSIONS.md) 갱신**
   - PIN `000000` = 관리자, `002531` = 개발자(김종필, 권한상 일반 사용자 + 명부 삭제권만 추가), 그 외 = 일반 회원.
5. **데이터 동기화 기능은 신중하게**
   - 계정(PIN)별 `directory`(전사원 명부 누적 카운트)는 **계정 독립 저장**이며, 계정 간 공유/병합 시도(v1.6.69~73)는
     카운트 초기화 사고를 일으켜 **전면 롤백(v1.6.74)** 됨. 유사한 "공유/병합" 기능은 다시 시도하기 전
     반드시 사용자와 설계를 재확인할 것.
   - 사용자가 회귀(데이터 초기화 등)를 보고하면 **추가 패치보다 즉시 직전 안정 버전으로 롤백**을 우선.

## 2. 작업 워크플로 (매 변경 시 필수)

```
1. 파일 수정 (app.js / index.html / style.css 등)
2. node --check app.js   ← 문법 오류 확인
3. sw.js 의 APP_VERSION 1 증가 (현재 1.6.75)
4. git add -A && git commit -q -m "..." && git push -q
```

- `sw.js`는 네트워크 우선 캐시 전략 + 업데이트 배너 트리거용 — 변경 시 버전 bump 누락하면 사용자에게 변경사항이 반영 안 됨.

## 3. 정산 계산 핵심 요약 (자세한 내용은 CALCULATION_SPEC.md 참조)

- 비용 카테고리: 행사비(EVENT) / 시설·장비(FACILITY) / 상품(PRIZE)
- 인당 행사비 = 행사비 합계 ÷ 참석자 수
- 인당 자부담 4단계 구간 (가/나/다/라, `AppState.rules`로 관리자가 한도·비율 조정 가능):
  - 가: ≤30,000원 → 0%
  - 나: ≤60,000원 → 20%
  - 다: ≤120,000원 → 60,000×20% + (초과분)×40%
  - 라: >120,000원 → 비용 - 85,000원
- 총 자부담 = round(인당 자부담 × 인원) → 사용자가 "총 자부담 금액" 직접 수정 가능 (`lastCalculatedSelfPay`),
  수정 시 `finalSelfPay`가 엑셀 K24/K25/K30, L25 안내문구에 반영됨.
- 엑셀(`lib/template.xlsx`) 수식 캐시 문제로 K4~K30 등 핵심 셀은 앱이 직접 값 계산해 덮어씀.

## 5. 주요 UI/기능 구조 (최근 추가/변경된 핵심 영역)

- **차트 탭**: 월별 추이 / 클럽별 예산 소진율 / 카테고리 비중 / 자부담 추이 — 전부 `maintainAspectRatio:false` +
  `.chart-canvas-wrap` (320px / 모바일 260px). 월별·소진율 차트는 "전체 선택" 체크박스 포함 다중 클럽 필터
  (`selectedOverallClubs`, `selectedUsageClubs`) 지원.
- **클럽별 정산이력 탭** (관리자):
  - "클럽 선택" 드롭다운 (전체 클럽=월별 그룹, 특정 클럽=목록)
  - "정산인 선택" 드롭다운: 선택된 클럽 기준 정산인이 **2명 이상일 때만 표시**, 1명이면 자동 숨김 (v1.6.75)
  - 각 항목에 "정산인: OOO" 표시 (`entry.creatorName`)
  - 삭제는 관리자만 가능 — globalHistory + 작성자 개인 이력 + 참석자 누적 카운트 동시 처리
- **정산 결과 화면**: "클럽 잔여 예산" 표시 (연간 예산 - 사용액, 음수면 경고색)
- **정산 이력** (일반 회원): 조회만 가능, 삭제 불가 (관리자에게 요청)
- **PWA 업데이트 배너**: `reg.waiting` 직접 확인 후 500ms 내 강제 reload
